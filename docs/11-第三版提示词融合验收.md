# 第三版提示词融合验收清单

## 1. 目的
本清单用于验证“第三版规则提示词融合”在真实链路中是否达到以下目标：
1. 输出不泄漏流程化元文本（如“执行1.1/调用Agent/方案A-B/复盘”）。
2. 信息不足时能稳定降级，不杜撰事实。
3. 关键模板保持统一约束，并可自动同步旧版默认副本。

## 3. 人工联调范围
建议按以下顺序联调，便于定位问题：
1. 灵感模式：`title -> description -> theme -> genre -> quick-generate`
2. 大纲链路：`OUTLINE_CREATE -> OUTLINE_CONTINUE -> OUTLINE_EXPAND_*`
3. 正文链路：`CHAPTER_GENERATION_* -> PARTIAL_REGENERATE -> AI_DENOISING`
4. 分析链路：`PLOT_ANALYSIS -> AUTO_CHARACTER_ANALYSIS -> AUTO_ORGANIZATION_ANALYSIS`

## 4. 每条链路的判定标准
每次联调建议固定一个输入样本，重复 3 轮，按以下标准判定：

1. 元文本隔离：
   期望：输出中不出现流程化话术与自我说明。
   失败示例：`执行1.1`、`调用Agent`、`作为AI助手`、`方案对比`。

2. 契约稳定：
   期望：JSON 结构与字段不变；正文类接口只返回正文内容。
   失败示例：多出解释段、markdown 包裹、字段缺失或类型变化。

3. 缺口降级：
   期望：信息不足时保持“目标->阻力->选择->后果”最小闭环，或返回保守结论。
   失败示例：凭空补设定、强行引入角色/组织、大段空话。

4. 连续性与可写性：
   期望：章节规划与正文可直接用于后续写作，不是抽象口号。
   失败示例：大量总结句、无动作推进、无决策后果。

## 5. 模板同步验收
验证步骤：
1. 选取一个用户，在数据库中保留旧默认模板副本（不要手改内容）。
2. 访问对应模板读取接口，触发 managed sync。
3. 检查模板内容是否升级到当前系统默认版本。

判定标准：
1. 旧默认副本被自动升级。
2. 用户自定义改写过的模板不会被覆盖。

## 6. 失败处理建议
出现失败时按以下顺序处理：
1. 先定位模板层还是运行时护栏层问题。
2. 若是模板问题，优先最小改动修正约束语句。
3. 若是运行时护栏误判，先补测试再调规则阈值。
4. 每次修正后至少重跑第 2 节自动化回归。

## 7. 发布门槛（建议）
满足以下条件再进入发布：
1. 自动化回归全绿。
2. 人工联调覆盖第 3 节全部链路且 3 轮无阻断失败。
3. 模板同步验收通过且无误覆盖用户自定义内容。
