# MuMuNovel v1.3.8 发布公告

我们发布了 `v1.3.8`。

本次版本重点围绕“更稳定地产出高质量小说内容”做了一轮关键打通：从项目默认创作偏好，到章节生成，再到章节质量验收，形成了更完整的创作闭环。对于长篇小说、多章节连续生成和后续质量修订，这一版会更顺手，也更可控。

## 本次更新亮点

### 1. 项目默认质量偏好正式贯通到生成链路
现在，项目级的质量偏好可以更自然地进入章节生成流程。

- 支持将项目默认的质量预设自动带入生成请求
- 支持将项目默认的额外质量要求一并透传
- 单章生成与批量生成保持一致，减少重复填写和参数漂移

这意味着：当你已经在项目层面设定好创作标准之后，后续章节生成会更贴近你的既定风格与质量要求。

### 2. 批量生成体验增强
批量生成弹窗新增了更完整的质量控制项，适合连续创作场景。

- 新增质量预设配置
- 新增额外质量要求输入
- 支持一键恢复项目默认设置

这样做的价值是：你可以在“沿用项目默认”与“本次临时微调”之间快速切换，既保证一致性，也保留灵活性。

### 3. 新增章节“质量验收”面板
剧情分析页新增质量验收视图，用于帮助你更快识别章节短板并进行有针对性的修订。

质量验收面板集中展示了：

- 综合得分
- 当前最弱项
- 修复建议
- 质量指标网格
- 优先处理方向
- 自动修订草稿摘要
- 质量画像摘要

相比之前需要在多处信息之间来回比对，现在可以在一个位置更集中地判断章节质量状态。

### 4. 分析链路与生成链路更加一致
这一版也对内部链路做了统一。

- 章节生成请求统一透传质量控制字段
- 分析页直接读取更完整的章节分析详情
- 质量建议、修订草稿和画像信息可以在同一页聚合查看

这能有效减少“生成时用了某套标准，但分析页看不到完整反馈”的割裂感。

## 适合谁升级

如果你有以下使用场景，这一版会明显更有帮助：

- 需要连续批量生成章节
- 希望项目级创作风格长期稳定
- 需要更快定位章节质量问题
- 想把“生成 → 诊断 → 修订”串成更顺的工作流
