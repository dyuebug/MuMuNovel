# 02 - 环境搭建

本文档将指导你从零开始搭建 MuMuAINovel 的开发环境。

## 系统要求

### 硬件要求
- **CPU**：双核及以上
- **内存**：8GB 及以上（推荐 16GB）
- **硬盘**：至少 10GB 可用空间
- **网络**：稳定的互联网连接（用于下载依赖和调用 AI API）

### 操作系统
- **Windows**：Windows 10/11（推荐使用 WSL2）
- **macOS**：macOS 10.15 及以上
- **Linux**：Ubuntu 20.04 及以上 / Debian / Fedora

## 必需软件安装

### 1. Git

**用途**：版本控制，克隆项目代码

**安装方法**：

**Windows**：
```bash

https://git-scm.com/download/win

```

**macOS**：
```bash

```

**Linux**：

## 后端环境配置

### 1. 创建 Python 虚拟环境

```bash
# 进入后端目录
cd backend

# 创建虚拟环境
python -m venv .venv

# 激活虚拟环境
# Windows (CMD)
.venv\Scripts\activate.bat

# Windows (PowerShell)
.venv\Scripts\Activate.ps1

# macOS/Linux
source .venv/bin/activate

# 验证虚拟环境
which python  # macOS/Linux
where python  # Windows
# 应该指向 .venv 目录下的 python
```

### 2. 安装 Python 依赖

```bash
# 确保虚拟环境已激活
pip install --upgrade pip

# 安装项目依赖
pip install -r requirements.txt

# 验证安装
pip list
```

### 3. 配置环境变量

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑 .env 文件
# Windows
notepad .env

# macOS/Linux
nano .env
# 或
vim .env
```

**必需配置项**：

```bash
# 数据库配置（开发环境使用 SQLite）
DATABASE_URL=sqlite+aiosqlite:///./data/mumuai.db

# AI 服务配置（至少配置一个）
OPENAI_API_KEY=your_openai_api_key_here
OPENAI_BASE_URL=https://api.openai.com/v1
DEFAULT_AI_PROVIDER=openai
DEFAULT_MODEL=gpt-4o-mini

# 本地账户登录
LOCAL_AUTH_ENABLED=true
LOCAL_AUTH_USERNAME=admin
LOCAL_AUTH_PASSWORD=your_secure_password_here

# JWT 密钥（随机生成）
SECRET_KEY=your_random_secret_key_here
```

**生成随机密钥**：
```bash
# Python 方式
python -c "import secrets; print(secrets.token_urlsafe(32))"

# OpenSSL 方式
openssl rand -base64 32
```

### 4. 初始化数据库

```bash
# 确保在 backend 目录下，虚拟环境已激活

# 运行数据库迁移
alembic upgrade head

# 验证数据库
ls data/
# 应该看到 mumuai.db 文件
```

## 使用 Docker Compose（推荐）

## 常见问题

### 1. Python 虚拟环境激活失败

**Windows PowerShell 报错**：
```
无法加载文件 .venv\Scripts\Activate.ps1，因为在此系统上禁止运行脚本
```

**解决方法**：
```powershell
# 以管理员身份运行 PowerShell
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### 2. pip 安装依赖失败

**错误**：网络超时或下载失败

**解决方法**：使用国内镜像源
```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 3. npm 安装依赖失败

**错误**：网络超时或下载失败

**解决方法**：使用国内镜像源
```bash
# 使用淘宝镜像
npm install --registry=https://registry.npmmirror.com

# 或永久设置
npm config set registry https://registry.npmmirror.com
```

### 4. Docker 启动失败

**错误**：端口被占用

**解决方法**：
```bash
# 查看端口占用
# Windows
netstat -ano | findstr :8000

# macOS/Linux
lsof -i :8000

# 修改 docker-compose.yml 中的端口映射
```

### 5. 数据库连接失败

**错误**：`sqlalchemy.exc.OperationalError`

**解决方法**：
1. 检查 `DATABASE_URL` 配置是否正确
2. 确保数据库服务已启动
3. 检查数据库用户名和密码
4. 使用 SQLite 进行开发（无需额外配置）
