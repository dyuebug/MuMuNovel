# 10 - 常见问题

本文档收集了 MuMuAINovel 开发和使用过程中的常见问题及解决方案。

## 环境搭建问题

### Q1: Python 虚拟环境激活失败

**问题描述**：
```
Windows PowerShell 报错：无法加载文件 .venv\Scripts\Activate.ps1，因为在此系统上禁止运行脚本
```

**解决方案**：
```powershell
# 以管理员身份运行 PowerShell
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser

# 或者使用 CMD
.venv\Scripts\activate.bat
```

### Q2: pip 安装依赖超时

**问题描述**：
```
ERROR: Could not install packages due to an EnvironmentError: HTTPSConnectionPool
```

**解决方案**：
```bash
# 使用国内镜像源
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 或永久配置
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q3: npm 安装依赖失败

**问题描述**：
```
npm ERR! network timeout
```

**解决方案**：
```bash
# 使用淘宝镜像
npm install --registry=https://registry.npmmirror.com

# 或永久配置
npm config set registry https://registry.npmmirror.com

# 或使用 yarn
yarn install
```

### Q4: Docker 端口被占用

**问题描述**：
```
Error starting userland proxy: listen tcp 0.0.0.0:8000: bind: address already in use
```

**解决方案**：
```bash
# 查看端口占用
# Windows
netstat -ano | findstr :8000

# Linux/macOS
lsof -i :8000

# 杀死占用进程
# Windows
taskkill /PID <PID> /F

# Linux/macOS
kill -9 <PID>

# 或修改 docker-compose.yml 中的端口映射
ports:
  - "8001:8000"  # 改为其他端口
```

## 数据库问题

### Q5: 数据库连接失败

**问题描述**：
```
sqlalchemy.exc.OperationalError: (psycopg2.OperationalError) could not connect to server
```

**解决方案**：

**检查 1：数据库服务是否启动**
```bash
# PostgreSQL
sudo systemctl status postgresql

# Docker
docker-compose ps postgres
```

**检查 2：连接配置是否正确**
```bash
# 检查 .env 文件
cat .env | grep DATABASE_URL

# 测试连接
psql -U mumuai -d mumuai_novel -h localhost
```

**检查 3：防火墙是否阻止**
```bash
# 允许 PostgreSQL 端口
sudo ufw allow 5432/tcp
```

### Q6: Alembic 迁移失败

**问题描述**：
```
alembic.util.exc.CommandError: Target database is not up to date
```

**解决方案**：
```bash
# 查看当前版本
alembic current

# 查看迁移历史
alembic history

# 如果版本不一致，先回滚
alembic downgrade base

# 重新升级
alembic upgrade head

# 如果还是失败，删除 alembic_version 表
psql -U mumuai -d mumuai_novel -c "DROP TABLE alembic_version;"
alembic stamp head
```

### Q7: 数据库连接池耗尽

**问题描述**：
```
sqlalchemy.exc.TimeoutError: QueuePool limit of size 30 overflow 20 reached
```

**解决方案**：

**方法 1：增加连接池大小**
```python
# backend/app/database.py
engine = create_async_engine(
    DATABASE_URL,
    pool_size=50,        # 增加核心连接数
    max_overflow=30,     # 增加溢出连接数
    pool_timeout=90,
)
```

**方法 2：检查连接泄漏**
```python
# 确保使用依赖注入获取数据库会话
@router.get("/items")
async def get_items(db: AsyncSession = Depends(get_db)):
    # 使用 db
    pass
    # 会话会自动关闭

# 避免手动创建会话
# ❌ 错误示例
async def bad_example():
    db = AsyncSession(engine)
    # 忘记关闭会话
```

**方法 3：监控连接使用**
```bash
# 访问健康检查端点
curl http://localhost:8000/health/db-sessions
```

## API 问题

### Q8: 401 Unauthorized 错误

**问题描述**：
```json
{"detail": "无效的认证凭证"}
```

**解决方案**：

**检查 1：Token 是否过期**
```javascript
// 前端检查 Token
const token = localStorage.getItem('token');
if (!token) {
  // 重新登录
}

// 解析 Token 查看过期时间
const payload = JSON.parse(atob(token.split('.')[1]));
console.log('Token 过期时间:', new Date(payload.exp * 1000));
```

**检查 2：Token 格式是否正确**
```javascript
// 正确格式
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

// 错误格式
Authorization: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...  // 缺少 Bearer
```

**检查 3：SECRET_KEY 是否一致**
```bash
# 确保前后端使用相同的 SECRET_KEY
cat .env | grep SECRET_KEY
```

### Q10: SSE 流式响应中断

**问题描述**：
```
EventSource 连接意外关闭
```

**解决方案**：

**检查 1：Nginx 配置**
```nginx
location /api {
    proxy_pass http://localhost:8000;

    # SSE 必需配置
    proxy_buffering off;
    proxy_cache off;
    proxy_read_timeout 86400s;
    proxy_send_timeout 86400s;

    # 保持连接
    proxy_http_version 1.1;
    proxy_set_header Connection "";
}
```

**检查 2：后端超时设置**
```python
# 使用 Uvicorn 时增加超时
uvicorn app.main:app --timeout-keep-alive 86400
```

**检查 3：前端重连机制**
```typescript
class SSEClient {
  private reconnectAttempts = 0;
  private maxReconnectAttempts = 5;

  connect(url: string, onMessage: Function) {
    const eventSource = new EventSource(url);

    eventSource.onerror = () => {
      eventSource.close();

      if (this.reconnectAttempts < this.maxReconnectAttempts) {
        this.reconnectAttempts++;
        setTimeout(() => this.connect(url, onMessage), 1000 * this.reconnectAttempts);
      }
    };
  }
}
```

## AI 服务问题

### Q11: OpenAI API 调用失败

**问题描述**：
```
openai.error.RateLimitError: Rate limit exceeded
```

**解决方案**：

**方法 1：实现重试机制**
```python
import asyncio
from tenacity import retry, stop_after_attempt, wait_exponential

@retry(
    stop=stop_after_attempt(3),
    wait=wait_exponential(multiplier=1, min=4, max=10)
)
async def call_openai_api(prompt: str):
    response = await client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": prompt}]
    )
    return response
```

**方法 2：使用队列限流**
```python
import asyncio
from asyncio import Semaphore

# 限制并发请求数
semaphore = Semaphore(5)

async def call_openai_with_limit(prompt: str):
    async with semaphore:
        return await call_openai_api(prompt)
```

**方法 3：切换到备用模型**
```python
async def call_ai_with_fallback(prompt: str):
    try:
        return await openai_provider.generate(prompt)
    except Exception as e:
        logger.warning(f"OpenAI 失败，切换到 Gemini: {e}")
        return await gemini_provider.generate(prompt)
```

## 前端问题

### Q13: 页面刷新后状态丢失

**问题描述**：
```
刷新页面后，用户登录状态丢失
```

**解决方案**：

**使用 Zustand persist 中间件**：
```typescript
import { create } from 'zustand';
import { persist } from 'zustand/middleware';

export const useAuthStore = create<AuthState>()(
  persist(
    (set) => ({
      token: null,
      user: null,
      login: (token, user) => set({ token, user }),
      logout: () => set({ token: null, user: null }),
    }),
    {
      name: 'auth-storage',  // localStorage key
    }
  )
);
```

### Q14: 路由跳转后页面不更新

**问题描述**：
```
使用 navigate() 跳转后，页面内容没有更新
```

**解决方案**：

**方法 1：使用 useEffect 监听路由参数**
```typescript
import { useParams, useNavigate } from 'react-router-dom';

export default function ProjectDetail() {
  const { id } = useParams();
  const [project, setProject] = useState<Project | null>(null);

  useEffect(() => {
    loadProject(id);
  }, [id]);  // 监听 id 变化

  const loadProject = async (projectId: string) => {
    const data = await projectApi.getProject(projectId);
    setProject(data);
  };

  return <div>{project?.name}</div>;
}
```

**方法 2：使用 key 强制重新渲染**
```typescript
<Route path="/projects/:id" element={<ProjectDetail key={id} />} />
```

### Q17: 列表渲染卡顿

**问题描述**：
```
渲染大量数据时页面卡顿
```

**解决方案**：

**方法 1：虚拟滚动**
```typescript
import { List } from 'react-virtualized';

function ProjectList({ projects }: { projects: Project[] }) {
  const rowRenderer = ({ index, key, style }) => (
    <div key={key} style={style}>
      <ProjectCard project={projects[index]} />
    </div>
  );

  return (
    <List
      width={800}
      height={600}
      rowCount={projects.length}
      rowHeight={100}
      rowRenderer={rowRenderer}
    />
  );
}
```

**方法 2：分页加载**
```typescript
function ProjectList() {
  const [page, setPage] = useState(1);
  const [projects, setProjects] = useState<Project[]>([]);

  const loadMore = async () => {
    const data = await projectApi.getProjects(page, 20);
    setProjects([...projects, ...data]);
    setPage(page + 1);
  };

  return (
    <InfiniteScroll
      dataLength={projects.length}
      next={loadMore}
      hasMore={true}
      loader={<Spin />}
    >
      {projects.map(p => <ProjectCard key={p.id} project={p} />)}
    </InfiniteScroll>
  );
}
```

**方法 3：使用 React.memo**
```typescript
const ProjectCard = memo(({ project }: { project: Project }) => {
  return <Card>{project.name}</Card>;
});
```

## 部署问题

### Q18: Docker 容器无法启动

**问题描述**：
```
docker-compose up 后容器立即退出
```

**解决方案**：

**检查 1：查看日志**
```bash
docker-compose logs backend
docker-compose logs frontend
```

**检查 2：环境变量配置**
```bash
# 检查 .env 文件是否存在
ls -la .env

# 检查环境变量是否正确加载
docker-compose config
```

**检查 3：端口冲突**
```bash
# 查看端口占用
docker-compose ps
netstat -tulpn | grep :8000
```

**检查 4：磁盘空间**
```bash
df -h
docker system df
docker system prune  # 清理未使用的资源
```

### Q19: Nginx 502 Bad Gateway

**问题描述**：
```
访问网站显示 502 错误
```

**解决方案**：

**检查 1：后端服务是否运行**
```bash
curl http://localhost:8000/health
docker-compose ps backend
```

**检查 2：Nginx 配置**
```bash
# 测试配置
sudo nginx -t

# 查看错误日志
sudo tail -f /var/log/nginx/error.log
```

**检查 3：SELinux 权限（CentOS/RHEL）**
```bash
# 临时关闭 SELinux
sudo setenforce 0

# 或配置 SELinux 允许 Nginx 连接
sudo setsebool -P httpd_can_network_connect 1
```

### Q20: SSL 证书问题

**问题描述**：
```
浏览器显示"您的连接不是私密连接"
```

**解决方案**：

**检查 1：证书是否过期**
```bash
# 查看证书信息
openssl x509 -in /etc/letsencrypt/live/your-domain.com/fullchain.pem -noout -dates

# 手动续期
sudo certbot renew
```

**检查 2：证书路径是否正确**
```nginx
# Nginx 配置
ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;
```

**检查 3：防火墙是否开放 443 端口**
```bash
sudo ufw allow 443/tcp
sudo ufw status
```
