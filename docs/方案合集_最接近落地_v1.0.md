## 1. PRD 正式文案 v1.0（摘要版）
定位: 小说创作操作系统（Novel Creation OS），以“作品状态更新”为核心，而非“正文堆叠”。

核心目标:
- 结构可控: 主线、支线、债务与节奏可观察、可调度。
- 人物可信: 角色与关系有状态、有变化、有一致性。
- 风格可守: 去 AI 味不靠提示词，靠声线与风格治理。
- 外部可经营: 读者心智、期待、留存纳入系统闭环。
- 长期可沉淀: 作者资产化与能力成长可累计。

系统分层:
- 定义层 → 状态层 → 判断层 → 生产层 → 质控层 → 经营层 → 沉淀层

核心模块:
- 作品定义中心
- 叙事状态中台
- 规则引擎
- 章节编排台
- 场景装配台
- 正文协作页
- 风格控制页
- 验收发布页
- 读者心智与连载经营
- 作者成长与表达资产

标准工作流:
- 定义作品 → 读取状态 → 规则诊断 → 编排章节 → 装配场景 → 协作写作 → 验收发布 → 外部经营 → 资产沉淀

风险治理:
- 系统过重、AI 抹平作者味、过度迎合读者、规则替代创作。

## 2. 页面原型说明书 v1.0（摘要版）
导航结构:
- 指挥台、作品定义、主线与弧线、角色与关系、章节编排、场景装配、正文协作、风格控制、验收发布、连载经营、读者心智、资产沉淀、作者成长。

关键页面结构（核心页）:
- 指挥台: 总览 + 今日任务 + 风险雷达 + 快速入口 + 时间线。
- 章节编排: 候选池 + 优先级矩阵 + 任务书详情。
- 场景装配: 素材库 + 场景轨道 + 诊断面板。
- 正文协作: 模式栏 + 任务约束栏 + 正文工作区 + 诊断与版本区。
- 风格控制: 作品基线 + 角色声线 + 漂移诊断 + 风格实验室。
- 验收发布: 结构 / 人物 / 情绪 / 风格 / 经营验收。
- 连载经营: 发布前/中/后策略与反馈回流。

状态流:
- 章节: 候选 → 评估 → 已规划 → 装配 → 写作 → 验收 → 正史/回炉/降级/归档。
- 场景: 灵感 → 定义 → 排入 → 预演 → 起稿 → 验收。

## 9. 叙事资源预算规则表与触发清单 v1.0（摘要版）
提供 10 类资源的触发表与复合病灶规则。

统一分级:
- 严重度 S1-S4
- 紧迫度 U1-U4
- 动作强度 A1-A4

章节核对 8 问:
- 本章主打资源是什么
- 是否超出 2 类高价值资源
- 是否高成本解决低等级问题
- 是否透支后续高潮资源
- 是否回收至少一笔情绪/信任欠账
- 是否新增承诺却无回报
- 是否把设定挤进人物/关系章
- 尾钩是否重复

## 12. 叙事规则 DSL 规格书 v1.0（摘要版）
语言目标:
- 声明式、状态优先、可编译、可解释、可覆写、可沉淀。

指令族:
- 定义、观察、预算、任务、场景、写作、验收、沉淀。

编译出口:
- 章节任务书、场景装配清单、正文提示包、验收清单、经营建议包、资产沉淀卡。

示例:
```dsl
CHAPTER #24 {
  GOAL "repair trust without confession"
  PUSH relation("Lin:Qin").trust +8
  SAVE twist AS high
  DELAY secret("QinIdentity") WITH clue("partial-confirm")
}
```

# 附: 术语对照
- 正史: 验收通过进入作品正式时间线。
- 债务: 伏笔、承诺、情绪欠账、秘密拖延。
- 资源: 可消耗或可透支的叙事资本。
- 预算: 对资源消耗的节奏与数量控制。
- 规则引擎: 诊断与推荐的决策内核。
- DSL: 统一描述作品、状态、资源、任务与验收的语言层。
