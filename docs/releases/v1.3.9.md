# MuMuNovel v1.3.9 发布公告

我们发布了 `v1.3.9`。

这一版是一次面向发布链路与版本治理的维护版本。它不追求新增创作功能，而是把“版本号、更新日志、发布说明、GitHub Release”这一整条链路补齐并对齐到当前仓库，确保后续围绕小说质量和生成能力的版本可以更稳定地交付给用户。

## 本次更新亮点

### 1. 新增一键发版脚本
现在可以直接使用仓库根目录下的发版脚本，统一完成以下步骤：

- 读取并校验版本号
- 校验 `CHANGELOG.md` 是否包含对应版本
- 读取 `docs/releases/<version>.md` 作为发布说明
- 校验本地与远端 tag 状态
- 推送分支与 tag
- 创建或更新 GitHub Release

这可以减少手工发版时的遗漏，尤其适合后续持续发布小说生成、质量分析与修订相关功能时复用。

### 2. 修正应用内发版元数据来源
这一版同步修正了应用内与发布相关的数据源：

- 版本检查改为读取当前仓库 `dyuebug/MuMuNovel` 的 Release 信息
- 更新日志抓取改为读取当前仓库的提交历史
- 应用内 GitHub 链接指向当前仓库，避免跳到旧仓库页面

这样一来，用户在应用里看到的版本信息、更新提示和更新日志，会与实际发布结果保持一致。

### 3. 统一版本元数据
本次还对齐了多个版本入口，减少后续发版时出现“页面版本号、环境变量、Release 版本不一致”的风险：

- 前端 `package.json` 版本升级到 `1.3.9`
- 后端默认配置版本升级到 `1.3.9`
- 环境示例文件中的 `APP_VERSION` 同步升级到 `1.3.9`
- README 版本徽章同步更新到 `1.3.9`

## 适合谁升级

如果你需要更稳定的版本管理与发布流程，这一版会更有帮助，尤其适合：

- 需要频繁发布和验证版本的维护者
- 需要在应用内查看最新版本与更新日志的使用者
- 需要保证 GitHub Release、版本号和变更日志一致的团队协作场景

## 本次验证情况

本次版本已完成以下验证：

- 前端构建通过
- 后端相关 Python 文件语法校验通过
- `release.ps1` 的完整 `v1.3.9` DryRun 通过
- DryRun 已验证 changelog、release notes、tag、push 与 GitHub Release 流程均可达
